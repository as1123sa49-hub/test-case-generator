# 測試案例自動產生工具

不需要安裝任何軟體，純前端網頁。上傳規格書 PDF → Gemini AI 自動產出結構化測試案例 → 匯出 CSV。

## 事前準備

需要一組 **Google Gemini API Key**，免費額度足夠日常使用。

1. 前往 [Google AI Studio](https://aistudio.google.com/app/apikey)
2. 點「Create API Key」
3. 複製 Key（格式：`AIzaSy...`）

## 使用方式

1. 用瀏覽器直接開啟 `index.html`
2. 貼上 Gemini API Key，點「💾 儲存」
3. 上傳規格書 PDF（支援拖曳）
4. 點「🚀 開始分析」
5. 等待 AI 完成（第 2 步有進度條與計時）
6. 產出後點「📊 匯出 CSV」

## 分析模式

| 模式 | 說明 | 使用時機 |
|------|------|--------|
| 全量產出 | 針對整份規格書產出所有案例 | 首次分析、完整規格 |
| 差異比對 | 只針對新舊版本的變動部分產出案例 | 版本迭代更新 |

差異比對需同時上傳**新版**與**舊版**規格書 PDF。

## 快取功能（自動記憶上次版本）

每次分析成功後，工具會自動將新版規格書的文字內容儲存在瀏覽器 IndexedDB 中。

下次開啟時，舊版規格書區塊會出現提示欄：

```
💾 上次分析版本：V1_spec.pdf  儲存於 2026/3/24 14:32
   [📋 用作舊版比對]  [🗑 清除]
```

點「📋 用作舊版比對」即可直接以快取版本做差異比對，**不需重新上傳舊版 PDF**。

## 附加分析模式

當畫面上已有分析結果時，再次點「開始分析」會彈出確認：

- **「➕ 附加到現有結果」** — 新案例合併到現有表格
- **「🗑 清空並重新分析」** — 清除現有案例，重新產出

## 結果表格說明

| 欄位 | 說明 |
|------|------|
| 編號 | 類型縮寫_類別縮寫_序號（如 `POS_ROOM_001`） |
| 測試類型 | 正面 / 負面 / 邊界 |
| 類別 | 大廳 / 房內 / 異常處理 / 後台 |
| 影響層級 | P0（核心）/ P1（一般）/ P2（UI/特效） |
| 功能模組 | 對應功能點，差異比對模式含 `[新增]` / `[變更]` 標注 |
| 規格來源 | 頁碼或章節編號 |
| 版本標籤 | 分析日期（格式：`YYYYMMDD_v1`） |

> ⚠️ 表格中標紅色 `—` 的欄位表示 AI 未正常產出，需手動補齊

## 資料夾結構

```
test-case-generator/
  index.html   ← 開啟此檔案使用
  app.js       ← 主邏輯（PDF 解析 + Gemini API + CSV 匯出）
  README.md    ← 本說明檔
```

## 傳給其他人使用

把整個 `test-case-generator/` 資料夾壓縮成 zip 傳給對方，解壓後直接用瀏覽器開啟 `index.html` 即可。
對方需自備 Gemini API Key，API Key 僅儲存於本機 `localStorage`，不會外傳。
